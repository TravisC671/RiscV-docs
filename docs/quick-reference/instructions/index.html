AArch 64 Instructions

adc - Add with carry

pg. 86

Syntax​

addc{s} Rd, Rn, Rm
{s} (optional) determines if the operation affects the PSTATE
Rd is the destination register where the sum is stored
Rn, Rm are the registers to be added together

Details​

This operation adds the carry bit from the PSTATE
Rd <- Rn + Rm + carry

add - Add

pg. 86

Syntax​

add{s} Rd, Rn, <Operand2>
{s} (optional) determines if the operation affects the PSTATE
Rd is the destination register where the sum is stored
Rn is the register to be added to
<Operand2> tbd

Details​

This operation adds two registers together, with some extra options
Rd <- Rn + Rx

adr - Form PC-Relative Address

pg. 78

Syntax​

adr Rd, <label>
Rd is the destination register where the 64-bit address will be stored
is the label where the address is stored,

Details​

This instruction is more efficient than ldr Rx, = Label because it can calculate the 64-bit address in one or two instructions without performing memory access. Has a range of +- 1 MB (adrp has a range of +- 4 GB)
Rd <- address of label

adrp - Form PC-Relative Address to 4 KB Page

pg. 78

Syntax​

adr Rd, <label>
Rd is the destination register where the 64-bit address will be stored
is the label where the address is stored,

Details​

adrp has a range of +-4 GB to the nearest 4 KB page (4096 bytes). The 21-bit immediate is shifted left by 12 bits and added to the PC.

The lower 12 bits of a label's address can be added to adrp to exactly address a label.

rd <- page address of <label>

and - Bitwise AND

pg. 88

Syntax​

and{s} Rd, Rn, <Operand2>
{s} (optional) determines if the operation affects the PSTATE
Rd is the destination register where the result is stored
Rn is the register that is anded
<Operand2> tbd

Details​

Uses the bitwise AND operation between the two registers

Rd <- Rn AND <Operand2>

asr - Arithmetic shift right

pg. 91

Syntax​

asr Rd, Rn, Rm
Rd is the destination register where the result is stored
Rn is the register that the shift is performed on
Rm is the register that is used as the shift amount

Details​

asr shifts the bits to the right by the amount of Rm, but it keeps the sign of the original value

Ex.
1011 0110 >> 2
Result:
1110 1101

b - Branch

pg. 70

Syntax​

b{<cond>} <target_label>
<cond> (optional) the condition that if true, the program will goto the target_label the list of conditions can be found here(TBD)
<target_label> can be any label in the current file or any label that is defined as .global(or .global) in any file that is linked

Details​

This puts the new address inside the PC address.

This can make conditional logic such as loop and if-then-else constructs.

bic - Bitwise Bit Clear

pg. 89

Syntax​

bic{s} Rd, Rn, <Operand2>
Rd is the destination register
Rn is the register that has the bic applied with the register in <Operand2>

Details​

The bit clear operation is pretty much like a mask that only keeps the bits in <Operand2> that are set to zero. The operation looks like this

Ex.
Rn = 1011 0110
op = 1111 1000 -> 0000 0111
Rd = 0000 0110

Rd <- Rn AND NOT(<Operand2>)

bl - Branch and Link

pg. 73

Syntax​

bl <target_address>
<target_address> can be any label in the current file or any label defined as .global(or .global) within the range of +-128MB

Details​

Saves the address of the next instruction in link register (x30), then load PC with new address

pg. 73

Syntax​

Xn contains the address of the next instruction

Details​

Saves the address of the next instruction in link register (x30), then loads PC with Xn

X30 <- PC + 4
PC <- Xn

Details​

cbnz - Compare and Branch if Nonzero

pg. 76

Syntax​

cbnz Rt, <label>
Rt The register that will be tested. if nonzero, it will go to the label
<label> can be any label in the file, must be word-aligned

Details​

Conditionally branches to the label if Rt is nonzero. has a range of +- 1MB

if Rt != 0 then
PC <- label
end if

end if

ccmn - Conditional Compare Negative

pg. 101

Syntax​

ccmn Rn, Rm|#<imm5>, #<nzcv>, <cond>
Rn register to be added with Rm|#<imm5> and set to PSTATE
Rm|#<imm5> either is a register or an unsigned 5-bit immediate
#<nzcv> oa value between 0-15, representing the NZCV PSTATE flags
<cond> condition that will be used for comparison.

Details​

Test and set PSTATE flags to a negated comparison or immediate

if <cond> then
PSTATE <- flags(Rn+Rm|imm5)
else
PSTATE <- <nzcv>
end if

Details​

Test and set PSTATE to a comparison or immediate

end if

mov - Move

pg. 90

Syntax​

Rn general purpose register

Details​

The mov instruction is an alias, chosen by the assembler, which is either orr, movz, movn, or add instructions. It facilitates choosing an immediate value or moving a register value to or from the stack pointer

Move an immediate to the destination register
Rd <- #<imm16> or #<pattern>

movk - Move wide with Keep

pg. 90

Syntax​

movk Rd, #<imm16>{, lsl #<shift>}
Rd is the destination register
#<imm16> is a 16-bit value

64-bit: 0, 16, 32, 48
32-bit: 0, 16

Details​

Any 64-bit value can be loaded with 4 movk instructions. If it is a pattern, it can be done with just one mov instruction, which is an alias for orr

sets only four bits and keeps the rest of the number

rd[shift + 15: shift] <- (imm16 << shift)

svc - Supervisor Call

pg. 106

Syntax​

svc <syscall_number>
<syscall_number> is encoded in the instruction. The operating system may determine which operating system service is requested

Details​

In Linux, <syscall_number> should always just be zero. The system call number is passed in x8 and six other parameters can be passed in on x0-x5.

On Linux, it is generally better to make system calls by using the corresponding C library function, rather than calling them directly from assembly. This is because the C library function may perform additional work before or after making the system call. For instance, the exit library function may invoke other functions to cleanly shut down the program before it performs the exit system call.

Syntax​

umaddl Xd, Wn, Wm, Ra
Xd is a 64-bit destination register
Wn is a 32-bit register and is the multiplicand
Wm is a 32-bit register and is the multiplier
Ra is a general register and is added to the product

Details​

Xd <- Ra+Wn*Wm

umnegl - Unsigned multiply negate long

pg. 93

Syntax​

umnegl Xd, Wn, Wm
Xd is a 64-bit destination register
Wn is a 32-bit register and is the multiplicand
Wm is a 32-bit register and is the multiplier

Details​

Xd <- -(Wn*Wm)

umsubl - Unsigned multiply long

pg. 93

Syntax​

umsubl Xd, Wn, Wm, Xa
Xd is a 64-bit destination register
Wn is a 32-bit register and is the multiplicand
Wm is a 32-bit register and is the multiplier
Ra is a general register and is subtracted by the product

Details​

Xd <- Ra-Wn*Wm

umulh - Unsigned multiply high

pg. 95

Syntax​

umulh Xd, Xn, Xm
Xd is the 64-bit destination register
Xn is the 64-bit multiplicand
Xm is the 64-bit multiplier

Details​

Calculate the upper 64 bits of the 128 product multiplication between 2 64-bit numbers. The lower 64 bits can be obtained using the mul instruction

Xd <- (Xn*Xm)[127:64]

umull - Unsigned multiply long

pg. 93

Syntax​

umull Xd, Wn, Wm
Xd is a 64-bit destination register
Wm is a 32-bit multiplicand register
Wn is a 32-bit multiplier register

Details​

Xd <- Wn*Wm
